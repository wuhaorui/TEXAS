# 2026-04-28 工作日志

## 修复德州扑克下注轮转逻辑

### 问题
用户报告：发牌前每个人下注一样多之后不发牌，每个人一直在点过牌。

### 排查过程
1. 最初使用 `phaseFirstPlayer` + `backToFirst` 判断一轮是否结束
2. 发现有边界情况会误判（特别是 preflop 阶段）
3. 尝试添加 `phaseFirstActionDone` 标志，但逻辑多余且有bug

### 最终修复方案
使用 `actedThisPhase` (Set) 跟踪本阶段已行动的玩家：

**核心逻辑**：
- 玩家行动 → 记录到 `actedThisPhase`
- 如果加注（bet/raise/allin 且提高了注额）→ 清除其他玩家的已行动标记
- 一轮结束条件：`所有active玩家都已行动` 且 `下注金额相等`
- 如果没有可行动玩家（`nextPlayer === -1`）→ 直接进入下一阶段

**关键代码改动**：
1. `startNewHand()` 和 `advancePhase()` 中初始化 `room.actedThisPhase = new Set()`
2. `playerAction` 中记录 `room.previousBet` 用于判断是否有加注
3. 加注时重置其他玩家的已行动标记：`room.actedThisPhase.clear(); room.actedThisPhase.add(playerIndex);`
4. 判断条件：`allMatched && allActed`

### 经验总结
- 判断一轮结束不能简单用"轮回到第一个玩家"
- 加注后其他玩家需要重新行动，必须重置他们的"已行动"状态
- 使用 Set 跟踪已行动玩家比用标志变量更可靠
