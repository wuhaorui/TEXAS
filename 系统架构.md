## 3. 数据模型

### 3.1 房间 (Room)

```javascript
{
    id: string,              // 房间号 (8位大写)
    players: Player[],       // 玩家数组
    deck: Card[],           // 牌堆
    communityCards: Card[],  // 公共牌 (最多5张)
    pot: number,            // 底池金额
    currentBet: number,     // 当前下注额
    phase: string,          // 游戏阶段
    dealer: number,         // 庄家索引
    currentPlayer: number,  // 当前行动玩家索引
    phaseBets: {},          // 本轮各玩家下注
    smallBlind: number,     // 小盲
    bigBlind: number,       // 大盲
    initialChips: number,   // 房间初始筹码
    gameStarted: boolean,    // 游戏是否已开始
    createdAt: number       // 创建时间戳
}
```

### 3.2 玩家 (Player)

```javascript
{
    id: string,         // Socket PlayerID (UUID)
    socketId: string,   // Socket ID
    name: string,       // 昵称
    chips: number,      // 剩余筹码
    hand: Card[],       // 手牌 (2张)
    currentBet: number,  // 本轮已下注
    folded: boolean,     // 是否弃牌
    allIn: boolean,      // 是否全下
    isHost: boolean,     // 是否房主
    ready: boolean       // 是否准备 (等待阶段)
}
```

### 3.3 扑克牌 (Card)

```javascript
{
    suit: string,   // 花色: ♠ ♥ ♦ ♣
    rank: string,   // 点数: 2-10 J Q K A
    value: number   // 牌值: 2-14
}
```

### 3.4 游戏阶段 (Phase)

```
waiting → preflop → flop → turn → river → showdown
  ↑                                         |
  └────────────── 下一手牌 ─────────────────┘
```

| 阶段 | 说明 |
|------|------|
| waiting | 等待玩家加入/准备 |
| preflop | 翻牌前 (发手牌) |
| flop | 翻牌圈 (发3张公共牌) |
| turn | 转牌圈 (发第4张公共牌) |
| river | 河牌圈 (发第5张公共牌) |
| showdown | 摊牌 (比大小) |

## 4. Socket 事件列表

### 4.1 客户端 → 服务端

| 事件 | 参数 | 说明 |
|------|------|------|
| `createRoom` | `{playerName, initialChips, baseUrl}` | 创建房间 |
| `joinRoom` | `{roomId, playerName}` | 加入房间 |
| `playerReady` | `{}` | 切换准备状态 |
| `startGame` | `{}` | 开始游戏 (仅房主) |
| `bet` | `{action, amount}` | 下注/跟注/加注/弃牌/全下 |
| `transferHost` | `{targetPlayerId}` | 转让房主 |
| `disconnect` | - | 断开连接 |

### 4.2 服务端 → 客户端

| 事件 | 数据 | 说明 |
|------|------|------|
| `playerJoined` | `{player, players[]}` | 玩家加入 |
| `playerReadyUpdate` | `{players[]}` | 准备状态更新 |
| `gameStarted` | `{dealer, players[], communityCards[], pot, ...}` | 游戏开始 |
| `gameUpdate` | `{players[], pot, currentBet, phase, ...}` | 游戏状态更新 |
| `phaseAdvanced` | `{phase, communityCards[], currentPlayer}` | 阶段推进 |
| `playerAction` | `{playerId, action}` | 玩家操作广播 |
| `gameEnd` | `{winner, hand, handName, pot}` | 单局结束 |
| `gameOver` | `{winner}` | 游戏结束 (有人破产) |
| `newHand` | `{dealer, players[], ...}` | 重新发牌 |
| `playerLeft` | `{playerId, playerName}` | 玩家离开 |
| `newHost` | `{hostId}` | 房主变更 |

### 4.3 bet 动作类型

| action | 说明 |
|--------|------|
| `fold` | 弃牌 |
| `check` | 过牌 |
| `call` | 跟注 |
| `bet` | 下注 |
| `raise` | 加注 |
| `allin` | 全下 |

## 5. 核心业务流程

### 5.1 创建房间

```
1. 房主输入昵称 + 选择筹码 (1000/2000/5000)
2. 点击"创建房间"
3. createRoom → 生成房间号 → 进入等待界面
4. 复制分享链接给好友
```

### 5.2 加入房间

```
1. 粘贴链接或输入房间号 + 输入昵称
2. 点击"加入"
3. joinRoom → 校验房间存在、未开始、未满、无重名
4. 进入等待界面
```

### 5.3 等待阶段

```
1. 所有玩家点击"准备"按钮
2. 玩家列表显示"已准备"徽章
3. 全部准备 + ≥2人 → 房主可点击"开始游戏"
```

### 5.4 下注流程

```
1. 小盲/大盲设置 (10/20)
2. 轮到你时显示操作按钮
3. 选择动作 (弃牌/过牌/跟注/下注/加注/全下)
4. 所有人行动完后自动进入下一阶段
5. 重复直到摊牌
```

## 6. 牌型大小 (从大到小)

| 牌型 | 说明 |
|------|------|
| 皇家同花顺 | A K Q J 10 同花色 |
| 同花顺 | 同花色的顺子 |
| 四条 | 四张相同点数的牌 |
| 葫芦 | 三条+一对 |
| 同花 | 五张同花色 |
| 顺子 | 五张顺子 |
| 三条 | 三张相同点数 |
| 两对 | 两个对子 |
| 一对 | 一对对子 |
| 高牌 | 无上述牌型 |

## 7. 关键实现要点

### 7.1 Socket 状态关联
```javascript
// 每个 socket 连接维护自己的 roomId 和 playerId
socket.roomId = roomId;    // 用户当前所在房间
socket.playerId = uuidv4(); // 用户唯一ID (整个会话不变)
```

### 7.2 房间存储
```javascript
// 内存存储 ( Railway 单实例部署才能保证数据不丢失 )
const rooms = {};
```

### 7.3 下注金额计算
```javascript
// 跟注金额 = 当前下注额 - 玩家已下注
toCall = room.currentBet - player.currentBet;

// 加注后新的当前下注额
room.currentBet = player.currentBet + amount;
```

### 7.4 发牌顺序
```
按索引交替发牌:
第1张 → 玩家0
第2张 → 玩家1
第3张 → 玩家0
第4张 → 玩家1
...
```

## 8. 部署配置

### 8.1 Railway 单实例 (重要!)
Socket.io 房间数据存储在内存中，必须单实例运行：

```toml
# railway.toml
[deployments]
replicas = 1
```

或 Railway Dashboard: Settings → Scaling → Replicas = 1

### 8.2 环境变量
```bash
PORT=3000  # Railway 自动设置
```
